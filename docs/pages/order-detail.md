# 배송 조회 (`/orders/[id]`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/orders/[id]/page.tsx`

## 한줄 요약
주문의 배송 상태를 4단계 타임라인으로 보여주는 페이지. 데모용 UI.

---

## 스크린 구성

1. **Header**
2. **뒤로가기 버튼** → `/orders`
3. **주문번호 + 주문일**
4. **배송 상태 타임라인 (4단계)**
   - 주문완료 → 배송준비 → 배송중 → 배송완료
   - 완료된 단계: 파란 그라데이션 아이콘 + 날짜 표시
   - 미완료 단계: 회색 점선 + 날짜 없음
   - 단계 사이: 연결선 (완료=그라데이션, 미완료=회색)
5. **배송 정보 카드** — 택배사, 운송장번호 (둘 다 하드코딩)
6. **주문 상품 목록** — 상품명, 수량, 단가, 소계
7. **결제 정보** — 총 결제금액
8. **데모 안내** — "배송 조회는 UI 데모입니다. 실제 배송 연동은 포함되지 않습니다."

---

## 사용자 흐름

- **어디서 오나**: 주문 내역(`/orders`)의 "배송 조회" 버튼
- **어디로 가나**:
  - "← 주문 내역" 버튼 → `/orders`

## 이렇게 만든 이유

- **4단계 타임라인 UI**: 실제 배송 추적 API가 없지만, 쇼핑몰에서 배송 상태를 어떻게 보여줄지 UI 시안으로 만들었어요.
- **가상 날짜 생성**: `getStepDates()` 함수가 주문일 기준으로 하루씩 더해서 각 단계 날짜를 만듦. 예: 주문일 3/25 → 배송준비 3/26 → 배송중 3/27 → 배송완료 3/28.
- **택배사 "CJ대한통운" 하드코딩**: 실제 데이터가 없으니까 한국에서 가장 흔한 택배사 이름을 넣었어요.
- **운송장번호 "-"**: 실제 번호 없음.

---

## 주의할 점

- `getCompletedStepCount(status)`: 현재 주문 상태에 따라 완료된 단계 수를 계산합니다.
  - "주문완료" → 1단계, "배송준비" → 2단계, "배송중" → 3단계, "배송완료" → 4단계
- 타임라인 아이콘에 Framer Motion `spring` 애니메이션 + `staggered delay`가 적용되어 순서대로 나타남.
- 데모 안내 문구가 하단에 노란 배너로 표시됩니다.
- 주문 상품 목록의 소계는 `unit_price × quantity`로 계산.
