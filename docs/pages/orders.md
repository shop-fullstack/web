# 주문 내역 (`/orders`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/orders/page.tsx`

## 한줄 요약
내 주문 목록을 상태별로 필터링해서 볼 수 있는 페이지.

---

## 스크린 구성

1. **Header**
2. **페이지 타이틀** — "주문 내역"
3. **상태 필터 탭** — 전체 / 주문완료 / 배송중 / 배송완료 (각각 건수 뱃지)
4. **주문 카드 목록** — 각 카드에:
   - 주문일 (YYYY.MM.DD)
   - 주문번호
   - 상품 요약 ("상품명 외 N건")
   - 총 결제금액
   - 상태 뱃지 (색상 다름)
   - "배송 조회" 버튼 → `/orders/[id]`
   - "재주문" 버튼 → `alert()` (미구현)
5. **빈 상태** — "주문 내역이 없습니다" + "상품 둘러보기" 버튼

---

## 사용자 흐름

- **어디서 오나**: 결제 성공 후 리다이렉트, 마이페이지의 "주문 내역" 링크, Header 등
- **어디로 가나**:
  - "배송 조회" → `/orders/[id]`
  - "상품 둘러보기" (빈 상태) → `/products`

---

## 사용하는 것들

| 구분 | 이름 | 파일 경로 |
|------|------|----------|
| 컴포넌트 | Header | `src/components/header.tsx` |
| 훅 | useOrders | `src/lib/queries.ts` |

### API 호출
- `useOrders(activeTab)` — 선택된 상태로 필터링된 주문 목록
- `useOrders("all")` — 전체 주문 (탭 건수 계산용으로 별도 호출)

---

## 이렇게 만든 이유

- **탭 건수를 위한 이중 쿼리**: 필터링된 결과와 전체 결과를 동시에 가져와서, 각 탭의 건수를 정확히 표시.
- **layoutId 탭 애니메이션**: `layoutId="activeTab"`으로 활성 탭 배경이 슬라이드. Framer Motion의 공유 레이아웃 애니메이션.
- **상태별 뱃지 색상**: 주문완료(파랑), 배송중(노랑), 배송완료(초록) — `getStatusBadgeClasses()` 함수에서 관리.
- **재주문 버튼**: 아직 `alert("재주문 기능은 준비 중입니다")`만 뜸. 미구현.

---

## 주의할 점

- 탭 상태명이 `"전체"`, `"주문완료"`, `"배송중"`, `"배송완료"` 한글 문자열이에요. API에 그대로 전달됨.
- `getOrderSummary()` 헬퍼: 주문 아이템 중 첫 번째 상품명 + "외 N건" 형식으로 요약 텍스트 생성.
- 날짜 포맷: `YYYY.MM.DD` (점으로 구분).
- 주문이 많아져도 페이지네이션은 없어요. 전체 목록을 한 번에 렌더링합니다.
- `"전체"` 탭일 때는 status 파라미터 없이 API 호출.
