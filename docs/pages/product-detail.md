# 상품 상세 (`/products/[id]`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/products/[id]/page.tsx`

## 한줄 요약
개별 상품의 가격, 스펙, 수량 선택 + 장바구니/바로구매 기능.

---

## 스크린 구성

1. **Header**
2. **브레드크럼** — 홈 > 상품 목록 > 카테고리 > 상품명
3. **메인 영역 (좌우 분할)**
   - **왼쪽**: 큰 이미지 (실제로는 카테고리 이모지 + 그라데이션 배경) + 하단 4개 썸네일
   - **오른쪽**:
     - 카테고리 뱃지
     - 상품명
     - 가격 박스 (단가, 박스가격, MOQ)
     - 수량 선택 (-, +, 직접 입력)
     - 총 금액 실시간 계산
     - "담기" / "바로 구매" 버튼 2개
4. **스펙 테이블** — 카테고리, 단가, 박스가격, MOQ, 원산지(있으면), 유통기한(있으면)

---

## 사용자 흐름

- **어디서 오나**: 상품 목록(`/products`)의 ProductCard 클릭, 홈 추천상품 클릭
- **어디로 가나**:
  - "담기" 버튼 → 장바구니에 추가 (페이지 이동 없음, 토스트 같은 피드백도 없음)
  - "바로 구매" 버튼 → 장바구니에 추가 + `/cart`로 이동
  - 브레드크럼 → `/`, `/products`

## 이렇게 만든 이유

- **이모지로 상품 이미지 대체**: 실제 상품 사진이 없어서 카테고리별 이모지 + 그라데이션으로 표현. 포트폴리오용이라 이미지 없이도 시각적으로 괜찮게.
- **"담기" vs "바로 구매" 분리**: B2B 쇼핑몰에서 흔한 패턴. 여러 상품을 담아두고 한번에 결제하거나, 바로 살 수 있게.
- **4개 썸네일**: 실제로는 전부 같은 이모지. 상품 이미지가 여러 장 있을 때를 대비한 UI 틀.
- **MOQ(최소주문수량) 강조**: B2B이기 때문에 MOQ가 중요. 가격 박스에서 눈에 띄게 표시.

## 주의할 점

- 수량은 1 이상이어야 합니다. -버튼 누르면 1에서 멈춤.
- 수량에 따라 하단 총 금액이 `price_per_box × quantity`로 실시간 계산됩니다.
- `origin`과 `expiry_info`는 optional 필드. 있을 때만 스펙 테이블에 표시.
- "담기" 버튼을 눌렀을 때 시각적 피드백이 없어요 (ProductCard에서는 "담았습니다"가 뜨지만, 상세 페이지에서는 없음). Header의 장바구니 뱃지 숫자가 바뀌는 걸로 확인 가능.
- 상품이 없으면 (잘못된 ID) 에러 화면 + "상품 목록으로" 버튼.
