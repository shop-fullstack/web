# 홈 (`/`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/page.tsx`

## 한줄 요약
비즈마트의 랜딩 페이지. 히어로 배너, 트렌드 미리보기, 카테고리, 추천 상품을 보여줍니다.

---

## 스크린 구성 (위에서 아래로)

1. **Header** — 공통 네비게이션
2. **Hero 섹션** — "소상공인을 위한 똑똑한 도매 플랫폼" 타이틀 + CTA 버튼 2개
3. **트렌드 배너** — 이번 주 인기상품 TOP 3 카드 (API에서 가져옴)
4. **카테고리 섹션** — 6개 카테고리 카드 (식자재, 소모품, 포장재, 뷰티용품, 인테리어, 기타)
5. **추천 상품 섹션** — 인기순 상품 4개 (ProductCard 컴포넌트 사용)
6. **Footer** — 저작권 표시 (홈에만 있음, 다른 페이지엔 없음)

---

## 사용자 흐름

- **어디서 오나**: 직접 접속, 로고 클릭, 로그인/회원가입 성공 후 리다이렉트
- **어디로 가나**:
  - Hero "상품 둘러보기" → `/products`
  - Hero "트렌드 확인하기" → `/trend`
  - 카테고리 카드 클릭 → `/products?category=식자재` (카테고리 필터 적용)
  - 추천 상품 카드 클릭 → `/products/[id]`
  - Header를 통해 다른 페이지로 이동

### API 호출
- `useProducts({ sort: "popular", limit: 4 })` — 인기 상품 4개
- `useTrendReport("weekly", 3)` — 주간 트렌드 TOP 3

## 이렇게 만든 이유

- **Hero에 그라데이션 + 블롭 애니메이션**: 첫인상을 화려하게 주기 위해서. 그리드 패턴 오버레이도 깔려있어요.
- **트렌드 배너를 홈에 넣은 이유**: 비즈마트의 핵심 차별점이 "AI 트렌드 분석"이라서, 홈에서부터 이 기능을 노출합니다.
- **카테고리 6개 하드코딩**: DB에서 가져오는 게 아니라 코드에 직접 써놨어요. 카테고리가 추가되면 여기도 수정해야 합니다.
- **추천 상품 4개만**: 홈은 맛보기이므로 4개만 보여주고, "전체 보기" 링크로 `/products`로 보냅니다.
- **Footer가 홈에만 있는 이유**: 다른 페이지는 컨텐츠가 충분해서 Footer를 안 넣었어요.

## 주의할 점

- 카테고리 목록이 하드코딩되어 있어요. 만약 백엔드에서 카테고리가 추가되면 이 페이지의 `categories` 배열, `products/page.tsx`의 카테고리 필터, `product-visuals.ts`의 매핑 — 3곳 모두 수정해야 합니다.
- 트렌드 데이터 로딩 실패 시 해당 섹션이 안 보이게 처리돼요 (에러 UI 없이 숨김).
- 추천 상품 로딩 실패 시 "다시 시도해주세요" 버튼이 나타납니다.
- Framer Motion의 `staggerChildren`으로 카드가 순서대로 등장하는데, 데이터가 늦게 오면 스켈레톤이 먼저 보이고 그 다음에 애니메이션이 돕니다.
- 변동 지표: `up`(▲초록), `down`(▼빨강), `same`(—회색), `new`(NEW보라뱃지)로 트렌드 카드에 표시됩니다.
