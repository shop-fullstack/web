# 상품 목록 (`/products`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/products/page.tsx`

## 한줄 요약
카테고리 필터, 검색, 정렬로 상품을 찾는 목록 페이지.

---

## 스크린 구성

1. **Header**
2. **페이지 타이틀** — "상품 목록" + 검색결과/필터 개수 표시
3. **레이아웃: 사이드바 + 메인**
   - **왼쪽 사이드바** (w-64 고정)
     - 검색 인풋 (돋보기 아이콘, 포커스 시 테두리 색상 변경)
     - 카테고리 필터 (6개 버튼, 토글 방식)
     - 정렬 옵션 (인기순, 가격↑, 가격↓, 최신순)
     - "필터 초기화" 버튼 (필터 활성화 시만 표시)
   - **오른쪽 메인** (상품 그리드)
     - 4열 그리드 (반응형: 1→2→3→4열)
     - ProductCard 컴포넌트 사용
     - 하단 페이지네이션

---

## 사용자 흐름

- **어디서 오나**: 홈의 "상품 둘러보기" 버튼, 카테고리 카드 클릭, Header "상품" 링크, 정기구독의 "추가하기" 버튼
- **어디로 가나**:
  - 상품 카드 클릭 → `/products/[id]`
  - 상품 카드 "담기" 버튼 → 장바구니에 추가 (페이지 이동 없음)

### API 호출
- `useProducts({ category, sort, search, page, limit: 20 })` — 필터/정렬/검색/페이지에 따라 상품 조회

---

## 이렇게 만든 이유

- **검색 디바운스 300ms**: 타이핑할 때마다 API 호출하면 너무 많으니까, 300ms 기다렸다가 한 번에 호출.
- **카테고리 토글 방식**: 같은 카테고리를 다시 클릭하면 해제. 라디오가 아니라 토글이에요.
- **사이드바 고정폭**: 반응형에서 모바일일 때는 상단으로 올라감 (flex 방향 변경).
- **shimmer 스켈레톤**: 로딩 중에 회색 박스가 번쩍번쩍 빛나는 효과. CSS gradient 애니메이션으로 구현.
- **페이지네이션 ellipsis**: 페이지가 많으면 `[1, ..., 5, 6, 7, ..., 20]` 이런 식으로 줄여서 표시.

---

## 주의할 점

- `useSearchParams`를 사용하지 않습니다. 홈에서 `/products?category=식자재`로 링크하지만, 이 페이지에서 URL 쿼리파라미터를 읽지 않아요. `useState`만 사용합니다. 즉, 홈에서 카테고리 링크로 올 때 자동 필터링이 안 될 수 있어요 (개선 포인트).
- 카테고리 목록 6개가 하드코딩. 홈 페이지, product-visuals.ts와 동기화 필요.
- 검색어를 지우면 즉시 전체 상품으로 돌아갑니다.
- `AnimatePresence mode="wait"`로 필터 변경 시 현재 그리드가 사라지고 새 그리드가 나타나는 전환 효과가 있어요.
- 한 페이지당 20개 고정 (LIMIT 상수).
