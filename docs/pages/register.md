# 회원가입 (`/register`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/register/page.tsx`

## 한줄 요약
이메일/비밀번호 + 사업자 정보를 입력해서 가입하는 페이지.

---

## 스크린 구성

1. **배경** — 로그인과 동일한 그라데이션 + 블롭
2. **회원가입 카드** — 글래스모피즘
   - 로고 + "회원가입" 제목
   - **계정 정보 섹션**: 이메일, 비밀번호, 비밀번호 확인
   - **구분선** ("사업자 정보" 라벨 포함)
   - **사업자 정보 섹션**: 사업자등록번호, 업종, 상호명, 대표자명
   - 에러 메시지 영역
   - "회원가입" 버튼
   - "로그인" 링크

---

## 사용자 흐름

- **어디서 오나**: 로그인 페이지의 "회원가입" 링크
- **어디로 가나**:
  - 가입 성공 → 자동 로그인 → `/` (홈)
  - "로그인" 링크 → `/login`

| 구분 | 이름 | 파일 경로 |

### API 호출
- `useRegister()` → POST `/auth/register` — 이메일, 비밀번호, 사업자등록번호, 업종, 상호명, 대표자명

---

## 이렇게 만든 이유

- **구분선으로 섹션 분리**: 계정 정보와 사업자 정보가 성격이 다르니까 시각적으로 분리.
- **업종 선택은 드롭다운**: 5개 선택지가 하드코딩 → `["카페/베이커리", "식당/외식업", "미용실/뷰티", "편의점/소매업", "네일샵/피부샵"]`
- **가입 성공 시 자동 로그인**: 가입 후 다시 로그인하게 하면 불편하니까, 응답의 토큰으로 바로 로그인 처리.
- **비밀번호 확인**: 프론트에서 일치 여부 확인. 불일치 시 "비밀번호가 일치하지 않습니다" 에러.

## 주의할 점

- 업종 목록은 하드코딩이에요. `trend/best/page.tsx`의 탭 목록과 동일한 5개인데, 한쪽만 바꾸면 안 맞을 수 있어요.
- 비밀번호 보기 토글이 2개 (비밀번호, 비밀번호 확인 각각).
- `EyeIcon` 컴포넌트가 이 파일 안에 로컬로 정의되어 있어요 (별도 컴포넌트 아님).
- 모든 필드가 필수입니다. 하나라도 빠지면 에러 메시지가 나와요.
- 이메일 중복 체크는 프론트에서 안 하고, 백엔드 에러로 처리합니다.
