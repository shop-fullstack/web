# 장바구니 (`/cart`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/cart/page.tsx`

## 한줄 요약
담은 상품 확인, 수량 조절, 결제 진행. API 호출 없이 Zustand 로컬 상태만 사용.

---

## 스크린 구성

1. **Header**
2. **페이지 타이틀** — "장바구니" + 상품 개수
3. **레이아웃 (좌우 분할)**
   - **왼쪽 (상품 목록)**
     - 각 아이템: 이모지 이미지, 상품명, 카테고리, 단가/박스가격, 수량 조절 (-, +), 삭제(X) 버튼
     - 삭제 시 `AnimatePresence mode="popLayout"`으로 부드러운 퇴장 애니메이션
   - **오른쪽 (주문 요약, sticky)**
     - 상품 금액 합계
     - 배송비 (무료)
     - 총 결제 금액
     - "결제하기" 버튼 → `/checkout`
4. **빈 장바구니 상태** — ShoppingCart 아이콘 + "마음에 드는 상품을 담아보세요" + "상품 둘러보기" 링크

---

## 사용자 흐름

- **어디서 오나**: 상품 상세의 "바로 구매" 버튼, Header의 장바구니 아이콘
- **어디로 가나**:
  - "결제하기" → `/checkout`
  - "상품 둘러보기" (빈 장바구니) → `/products`
  - 상품명 클릭 → 없음 (링크 아님, 개선 포인트)

---

## 이렇게 만든 이유

- **API 없이 로컬 저장**: 장바구니를 서버에 저장하려면 로그인이 필수인데, 비로그인 상태에서도 담을 수 있게 하려고 로컬로 처리.
- **Zustand persist**: localStorage에 `"bizmart-cart"` 키로 자동 저장. 브라우저 닫았다 열어도 유지됨.
- **sticky 주문 요약**: 스크롤해도 결제 금액이 항상 보이게. 쇼핑몰에서 흔한 패턴.
- **삭제 애니메이션**: `popLayout` 모드로 아이템이 쏙 빠지면서 아래 아이템이 올라오는 효과.

---

## 주의할 점

- 수량을 0 이하로 내리면 `updateQuantity()`에서 자동으로 아이템 삭제됩니다.
- 배송비는 항상 0원으로 표시 (무료 배송). 하드코딩.
- 장바구니 아이템에서 상품 상세로 가는 링크가 없어요. 상품명은 텍스트만.
- `totalAmount()`는 `items.reduce(sum of price_per_box * quantity)`.
- 장바구니가 비면 "결제하기" 버튼 대신 빈 상태 UI가 표시됩니다.
