# 로그인 (`/login`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/login/page.tsx`

## 한줄 요약
이메일 + 비밀번호로 로그인하는 페이지. 글래스모피즘 UI.

---

## 스크린 구성

1. **배경** — 그라데이션 + 블롭 애니메이션 (장식용)
2. **로그인 카드** — 글래스모피즘 효과 (backdrop-blur)
   - 로고 + "로그인" 제목
   - 이메일 입력
   - 비밀번호 입력 (눈 아이콘으로 보기/숨기기 토글)
   - 에러 메시지 영역
   - "로그인" 버튼
   - "회원가입" 링크

---

## 사용자 흐름

- **어디서 오나**: Header의 "로그인" 버튼, 401 에러 리다이렉트, 직접 접속
- **어디로 가나**:
  - 로그인 성공 → `/` (홈)
  - "회원가입" 링크 → `/register`

---

## 사용하는 것들

| 구분 | 이름 | 파일 경로 |
|------|------|----------|
| 훅 | useLogin | `src/lib/queries.ts` |
| 스토어 | useAuthStore | `src/store/auth-store.ts` |

### API 호출
- `useLogin()` → POST `/auth/login` — `{ email, password }`

---

## 이렇게 만든 이유

- **Header가 없는 이유**: 로그인 전이니까 네비게이션이 필요 없어요. 깔끔한 집중 화면.
- **글래스모피즘 UI**: 회원가입 페이지와 동일한 디자인. 모던한 느낌을 주려고.
- **비밀번호 보기 토글**: UX 개선. EyeIcon / EyeOffIcon으로 전환.
- **에러 처리 방식**: Axios 에러에서 `response.data.message`를 추출해서 보여줘요. 네트워크 에러면 "로그인에 실패했습니다" 기본 메시지.

---

## 주의할 점

- 로그인 성공 시 `useAuthStore.login(token, user)`를 호출해서 Zustand + localStorage 모두 갱신합니다.
- `auth.ts`의 `setToken()`은 localStorage와 cookie 양쪽에 저장해요.
- 이메일/비밀번호 빈 칸 체크는 프론트에서 하지만, 포맷 검증은 안 합니다.
- 로딩 중일 때 버튼에 Loader2 스피너가 돌아가고 비활성화됩니다.
