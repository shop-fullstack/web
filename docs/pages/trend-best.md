# 업종별 베스트셀러 (`/trend/best`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/trend/best/page.tsx`

## 한줄 요약
업종(카페, 식당 등)을 선택하면 해당 업종의 인기 상품 TOP 10을 보여주는 페이지.

---

## 스크린 구성

1. **Header**
2. **페이지 타이틀** — "업종별 베스트셀러" + 설명
3. **업종 탭 5개** — 카페/베이커리, 식당/외식업, 미용실/뷰티, 편의점/소매업, 네일샵/피부샵
4. **TOP 10 랭킹 리스트**
   - 순위 뱃지 (1~3등 컬러, 4등부터 회색)
   - 상품명
   - 카테고리 뱃지
   - 주문수
   - "담기" 버튼
5. **"함께 많이 구매한 상품" 섹션** — 5열 카드 그리드
   - ranking의 5~10번째 아이템을 재활용

---

## 사용자 흐름

- **어디서 오나**: Header의 "트렌드" 링크, 트렌드 리포트에서 직접 URL 접근
- **어디로 가나**:
  - 탭 전환 → 같은 페이지 (데이터만 변경)
  - "담기" 버튼 → 장바구니에 추가... 하는 것 같지만 **실제로는 미구현**

---

## 이렇게 만든 이유

- **업종별 분리**: 비즈마트의 핵심 차별점. 카페 사장님과 식당 사장님은 필요한 상품이 다르니까 업종별로 트렌드를 보여줌.
- **"함께 많이 구매한 상품"**: 별도 API가 없어서 ranking의 5~10번째를 잘라서 재활용. 데이터 활용 효율.
- **탭 애니메이션**: `layoutId="activeTabBg"`로 활성 탭 배경이 슬라이드. `AnimatePresence mode="wait"`로 리스트 전환.
- **RankNumber 로컬 컴포넌트**: 이 파일 안에서만 사용. 1등 금색, 2등 은색, 3등 동색, 나머지 회색.

---

## 주의할 점

- **"담기" 버튼이 미구현이에요!** `<button>` 태그만 있고 `onClick`에 장바구니 추가 로직이 없습니다. 클릭해도 아무 일도 안 일어남.
- 업종 목록 5개가 하드코딩. `register/page.tsx`의 업종 드롭다운과 동일한 목록인데, 한쪽만 바꾸면 안 맞아요.
- "함께 많이 구매한 상품"은 `ranking.slice(5, 10)`으로 5~10위를 가져옴. 10개 미만이면 비어있을 수 있음.
- 탭 전환 시 `AnimatePresence`의 `mode="wait"`로 이전 리스트가 사라진 후 새 리스트가 나타남.
- 로딩 중: Loader2 스피너, 에러 시: 에러 메시지 표시.
