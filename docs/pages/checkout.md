## 이렇게 만든 이유

- **Mock 결제**: 포트폴리오 프로젝트라 실제 PG를 붙이지 않았어요. 버튼 누르면 바로 주문 완료.
- **배송지 하드코딩**: 데모라서 배송지를 입력받지 않고 고정값 사용. `"서울특별시 마포구 합정동 123-45"`.
- **배송일 하드코딩**: `"2026-04-05"`. 실제라면 달력 선택 UI가 필요.
- **결제수단 3개**: 카드, 간편결제, 무통장입금 — 실제 동작은 없고 UI만. 선택한 값을 서버에 보내지도 않습니다.
- **동의 체크 필수**: 체크 안 하면 결제 불가. "주문 내용을 확인하였으며, 결제에 동의합니다" 문구.

## 주의할 점

- 결제 성공 후 3가지 동작이 순서대로 일어납니다:
  1. `useCartStore.clear()` — 장바구니 비움
  2. `queryClient.invalidateQueries(["orders"])` — 주문 목록 캐시 갱신
  3. `router.push("/orders")` — 주문 내역으로 이동
- `queryClient`를 `useQueryClient()`로 직접 가져와서 사용합니다.
- `is_cold`는 항상 `false` 고정. 냉장 배송 옵션 UI가 없음.
- 장바구니가 비어있는 상태에서 `/checkout`에 직접 접근하면 빈 주문이 생길 수 있어요 (가드 없음, 개선 포인트).
- 에러 발생 시 Axios 에러에서 메시지 추출해서 빨간색으로 표시.
