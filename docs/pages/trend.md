# 트렌드 리포트 (`/trend`)

> 마지막 확인: 2026-03-30 | 파일: `src/app/trend/page.tsx`

## 한줄 요약
주간 인기 상품 TOP 10 + 카테고리별 분포 차트를 보여주는 대시보드.

---

## 스크린 구성

1. **Header**
2. **페이지 타이틀** — "트렌드 리포트" + "이번 주 인기 상품 분석" 설명
3. **통계 카드 4개** (상단 가로 배치)
   - 인기 상품 1위 (이름 표시)
   - 급상승 상품 (change="up"인 첫 번째)
   - 카테고리 수 (ranking에서 unique 카테고리 개수)
   - 누적 구매자 (전체 order_count 합계)
4. **2열 레이아웃**
   - **왼쪽: TOP 10 테이블**
     - 순위, 상품명, 카테고리, 주문수, 변동
     - 1등: 금색 뱃지, 2등: 은색, 3등: 동색 (RankBadge 컴포넌트)
     - 변동 표시: ▲초록, ▼빨강, —회색, NEW보라
   - **오른쪽: 카테고리별 분포**
     - 가로 바 차트 (막대 너비 = 해당 카테고리 비율)
     - 6개 그라데이션 색상이 순환

---

## 사용자 흐름

- **어디서 오나**: 홈의 "트렌드 확인하기" 버튼, Header의 "트렌드" 링크
- **어디로 가나**:
  - 특별한 내부 링크 없음 (정보 확인 페이지)
  - Header를 통해서만 다른 페이지로 이동

## 이렇게 만든 이유

- **4개 통계 카드**: 한눈에 핵심 지표를 보여주려고. 대시보드의 기본 패턴.
- **카테고리 분포를 ranking 데이터에서 계산**: 별도 API 없이, TOP 10 데이터의 카테고리를 세서 분포를 만들었어요. API 호출을 줄이려는 의도.
- **RankBadge 로컬 컴포넌트**: 이 파일 안에서만 쓰이는 컴포넌트. 1/2/3등에 금/은/동 그라데이션 적용.
- **바 차트를 직접 구현**: 차트 라이브러리(Chart.js 등) 안 쓰고 div 너비로 직접 만들었어요. 의존성 줄이려고.
- **barGradients 순환 배열**: 6개 그라데이션 색상이 `index % 6`으로 순환. 카테고리가 늘어도 색이 부족하지 않음.

---

## 주의할 점

- "급상승 상품"은 ranking에서 `change === "up"`인 첫 번째 아이템. 없으면 "-" 표시.
- 카테고리 분포 계산 로직: `ranking.reduce()`로 카테고리별 count → 비율 계산 → 내림차순 정렬.
- 바 차트 너비는 `(count / maxCount) * 100%`로 최대값 기준 비율.
- 로딩 중: Loader2 스피너가 페이지 중앙에 표시.
- 에러 시: "데이터를 불러올 수 없습니다" 메시지.
- `period` 파라미터가 "weekly"로 고정되어 있어요. 월간/일간 전환 UI는 없습니다.
